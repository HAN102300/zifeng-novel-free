书源管理系统问题分析与解决方案

  📋 当前架构分析

  您的项目已经实现了Legado3书源格式的基本支持，包括：
  - ✅ API类型书源: 基于JSON接口的书源
  - ✅ HTML类型书源: 基于CSS选择器的网页解析
  - ⚠️ JS脚本类型: 基本实现但存在严重缺陷

  🚨 主要问题诊断

  1. 登录系统缺陷

  startBrowserAwait: (url, title) => {
    console.log("[JS-BROWSER-AWAIT]", url, title);

  - 浏览器启动功能仅打印日志，无实际功能
  - 无法处理复杂的登录流程（验证码、滑块、短信验证等）
  - 登录状态管理不完善，无法持久化cookies/session

  2. JS脚本执行环境缺陷

  🛠️ 3. 完整的登录UI处理

  创建动态登录界面：
  const createLoginInterface = (source) => {
    const loginUi = JSON.parse(source.loginUi || '[]');

    return (
      <Modal title={`${source.bookSourceName} - 登录`}>
        <div className="login-form">
          {loginUi.map((item, index) => {
            switch (item.type) {
              case 'text':
                return <Input key={index} placeholder={item.name} />;
              case 'password':
                return <Input.Password key={index} placeholder={item.name} />;
              case 'button':
                return (
                  <Button
                    key={index}
                    onClick={() => handleLoginAction(item.action, source)}
                  >
                    {item.name}
                  </Button>
                );
              case 'checkbox':
                return (
                  <Checkbox key={index}>
                    {item.name}
                  </Checkbox>
                );
              default:
                return null;
            }
          })}
        </div>
      </Modal>
    );
  };

  // 处理登录动作
  const handleLoginAction = async (action, source) => {
    if (action.startsWith('http')) {
      // 外部链接
      window.open(action, '_blank');
    } else if (action.includes('startBrowser')) {
      // 启动浏览器登录
      const result = await javaShim.startBrowserAwait(action.split("'")[1]);
      if (result) {
        setSourceLoginStatus(source.bookSourceUrl, {
          loggedIn: true,
          cookies: result
        });
      }
    } else {
      // JS函数调用
      try {
        const func = new Function('java', 'source', action);
        await func(javaShim, source);
      } catch (e) {
        console.error('登录脚本执行失败:', e);
      }
    }
  };

  🛠️ 4. 书源分类管理增强

  // 书源类型检测和分类
  const detectSourceType = (source) => {
    const type = {
      api: false,
      html: false,
      javascript: false,
      loginRequired: false
    };

    // 检测API类型
    if (source.searchUrl && source.searchUrl.includes('{{key}}')) {
      type.api = true;
    }

    // 检测HTML类型
    if (source.ruleSearch && source.ruleSearch.bookList.includes('@')) {
      type.html = true;
    }

    // 检测JS类型
    if (source.jsLib || source.exploreUrl.includes('@js:')) {
      type.javascript = true;
    }

    // 检测登录需求
    if (source.loginUrl || source.loginUi) {
      type.loginRequired = true;
    }

    return type;
  };

  // 根据类型提供不同的管理界面
  const renderSourceByType = (source) => {
    const type = detectSourceType(source);

    if (type.javascript && type.loginRequired) {
      return <AdvancedSourceCard source={source} />;
    } else if (type.api) {
      return <ApiSourceCard source={source} />;
    } else if (type.html) {
      return <HtmlSourceCard source={source} />;
    }
  };
